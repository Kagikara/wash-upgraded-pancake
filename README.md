# wash-upgraded-pancake
*！给量化人提供最真实的可用于策略的数据！*

L1和L2股票数据清洗程序，提供用户客制化清洗需求选择。

支持配置文件记录问题处理模块扩展，提高可处理问题的广泛性。

工作流：

*1. LCI 配置导入层*

支持从配置文件导入清洗策略
规则集可配置（启用/禁用/优先级）
处理方式可配置（同一问题多种处理策略）
支持保存/加载配置版本（回滚用）

*2. 数据读取（Loader）*

CSV / Parquet 输入
字段映射（schema 支持）
解析错误记录到 load_errors
输出标准化 Record 列表

*3. 审查/校验（Validator）*

规则插件化（Rule 接口 + 注册表）
问题类别覆盖六大类（可扩展）
输出 Issue 列表，不直接修改数据
支持按类别汇总统计（供后续可视化）

*4. 人工核对与可视化输出（Review Stage）*

所有 Issues 列表（可过滤：股票/日期/类型）
问题分布概况图表：
时间分布（哪几天异常密集）
类型分布（哪类问题最多）
修改预览：
自动修改 vs 需人工审查
修改前后值对比（preview）
人工选择处理开关：
对某些 Issue 标记 “忽略/保留/处理”

*5. 清洗处理（Cleaner / Policy Engine）*

只对“允许处理”的 Issue 执行修复
处理规则可扩展（与问题解耦）
支持默认策略 + 自定义策略
保留原始数据不覆盖

*6. 输出层（Writer）*

输出清洗后的新数据文件
支持 CSV / Parquet
输出路径与版本号绑定

*7. Audit Log*

记录：
原值 / 新值 / 问题类型 / 处理动作
问题类别 / 触发规则
是否自动处理 or 人工处理
时间戳
运行性能指标（新增）
总耗时
吞吐量（行/秒）
各规则耗时分布

*8. LLM 报告模块*

输入：Issue 汇总 + Audit Log
输出：自然语言清洗报告
问题概况
处理统计
主要风险提示

*9. 版本控制*

每次执行记录一个版本快照
快照内容：配置文件 + 输出文件 + Audit Log
支持回滚到历史版本
LCI 可选择“指定版本导出”
